\chapter{Background}
\label{cha:bg}
This chapter establishes the conceptual vocabulary required to
understand, motivate, and situate the contributions of this project.
The subject is the regret-based analysis of randomized search
heuristics on pseudo-Boolean black-box optimization problems. Four
bodies of knowledge bear on this subject: the structure of the
optimization problems themselves, the algorithms used to solve them,
the performance measures used to evaluate those algorithms, and the
regret framework imported from online learning to address the
limitations of classical measures. These are introduced in that order.

\section{Computational Problems}

\subsection{Optimization Problems}

An \textit{optimization problem} is a formal specification of the task of
finding a candidate solution that is optimal with respect to a given
objective function. The optimization can be minimizing or maximizing;
this project adopts the latter convention. Given a set of candidate solutions
$\mathcal{X}$ and an objective function $f : \mathcal{X} \to \mathbb{R}$,
the goal is to identify
\[
  x^* \in \operatorname*{arg\,max}_{x \in \mathcal{X}} f(x).
\]

A further distinction concerns the role of randomness in the problem
formulation. In a \textit{deterministic} formulation, both the problem
instance and every algorithm operating on it are fully specified without
any probabilistic component. In a \textit{stochastic} formulation, the
instance itself, the objective evaluations, or the algorithm may involve
randomness. \textit{Noisy} objective functions, where repeated evaluation
of the same candidate solution may return different values, are a canonical
example and arise naturally in simulation-based and real-world
settings~\cite{liu_primer_2020}. All problems studied in this project are
deterministic, though the algorithms applied to them are randomized; this
distinction is elaborated in Section~\ref{subsec:algorithms}.

\subsection{Search Space}
\subsubsection{Structure of the Search Space}
% Finite vs. infinite spaces.
% Discrete vs. continuous domains.
% Combinatorial vs. geometric domains.

The \textit{search space} $\mathcal{X}$ is the set of all candidate
solutions considered
by an algorithm. Search spaces may be \textit{finite} or \textit{infinite}, and
\textit{discrete} or \textit{continuous}.

Discrete combinatorial search spaces, in particular the Boolean
hypercube $\{0,1\}^n$, are central to the theoretical analysis of
evolutionary algorithms (\acrshort{ea}) and other randomized search
heuristics~\cite{droste2002analysis,hoos2005sls} used in theoretical
\acrshort{bbo} analysis.
Continuous spaces $\mathbb{R}^d$ arise in numerical
optimization~\cite{nocedal2006numerical}, and mixed spaces containing
both discrete and continuous variables occur in practice but are
considerably harder to
analyze~\cite{audet2017derivative}.

The structure of the search space, and in particular how neighboring
solutions relate to
one another, fundamentally influences the difficulty of optimization.
This is encoded by
a \textit{neighborhood function} $\mathcal{N} : \mathcal{X} \to
2^{\mathcal{X}}$,
assigning to each candidate solution the set of solutions reachable
in a single step~\cite{hoos2005intro}.
For the Boolean hypercube $\{0,1\}^n$, the standard neighborhood consists of all
strings at Hamming
distance one\footnote{\textit{Hamming distance} between two
  bitstrings of equal length is the number of positions at which the
corresponding bits differ}.

This project is concerned exclusively with \textit{pseudo-Boolean
optimization}: problems in which the search space is the Boolean hypercube
$\mathcal{X} = \{0,1\}^n$ and the objective function takes non-negative
real values. A candidate solution is represented as a \textit{bitstring},
a sequence of bits $x = (x_1, \ldots, x_n) \in \{0,1\}^n$, and the
integer $n$ is referred to as the \textit{problem dimension}. The
hypercube has cardinality $2^n$, so exhaustive enumeration of all
candidate solutions is intractable for any practically relevant $n$.\footnote{%
  \begin{remark}[Objective function convention]
    Without loss of generality, $f$ is taken to be non-negative throughout
    this project. Any objective function with co-domain
    $\mathcal{T} \subseteq \mathbb{R}$ can be shifted via the affine map
    \[
      g: \mathbb{R} \to \mathbb{R}_0^+, \qquad g(x) = x - \min_{y \in
      \mathcal{X}} f(y),
    \]
    which is well-defined whenever $f$ attains its minimum on $\mathcal{X}$
    and preserves the ordering of all objective values by construction.
  \end{remark}
}

\subsubsection{Constraints and Feasibility}
% Feasible region definition.
% Constrained vs. unconstrained problems.

In a \textit{constrained} problem, the \textit{feasible region}
$\mathcal{F} \subseteq \mathcal{X}$ is defined by conditions a valid
solution must satisfy; an \textit{unconstrained} problem places no
restriction beyond membership in $\mathcal{X}$~\cite{nocedal2006numerical}.
At times, the constraints themselves can be black-box.
All pseudo-Boolean problems studied in this project are unconstrained.

\subsection{Objective Functions}

The geometric structure of the objective function determines both
which algorithms are
applicable and what convergence guarantees are attainable. A central organizing
principle is that access to higher-order analytical information about
$f$, i.e., gradients,
curvature, or structural properties such as convexity, generally
enables stronger
theoretical guarantees and more efficient algorithms; restricting this access,
as in the black-box setting, correspondingly limits what can be guaranteed.

\subsubsection{Oracle-Based Computation}
\label{subsubsec:oraclecomp}
% Definition of oracle access.
% Query-response interaction model.

An \textit{oracle} is an abstract computational device that answers
queries about a
problem instance. In optimization, the most common oracle is the
\textit{function
evaluation oracle}, which, given a query point $x \in \mathcal{X}$, returns the
objective value $f(x)$ (or a noisy estimate thereof). The algorithm
interacts with the
oracle in a \textit{query-response} model: it submits a query and
receives a response
before deciding on the next query.

\subsubsection{Zeroth-, First-, and Second-Order Methods}
\label{subsubsec:order-methods}

Optimization algorithms are classified by the order of analytical
information they require. \textit{Zeroth-order} methods use only
evaluations of $f(x)$; \textit{first-order} methods additionally
access the gradient $\nabla f(x)$; \textit{second-order} methods
further access the Hessian $\nabla^2 f(x)$. Access to higher-order
information generally enables stronger convergence guarantees and more
efficient algorithms~\cite{nocedal2006numerical}, but requires
corresponding structural
assumptions about $f$. Formal definitions
and convergence rates for each tier are given in
Appendix~\ref{app:oracle-tiers}.

This project operates exclusively in the zeroth-order regime.
When the internal structure of $f$ is inaccessible or analytically
intractable, function evaluations are the only information available.
Hence, these problems are often referred to as \textit{\acrfull{dfo}}
problems. These conditions lay ground for the black-box model formalized in
Section~\ref{sec:bbo}.

\section{Black-Box Optimization}
\label{sec:bbo}
The project is concerned with algorithms that interact with the
objective function exclusively through a query oracle, the
\textit{black-box} setting. This section formalizes that model,
establishes the complexity measures used to evaluate algorithms
within it, and introduces the benchmark problems and algorithm
families that are studied throughout the rest of the report.

\subsection{Algorithms}
\label{subsec:algorithms}

An \textit{algorithm} is a finite, unambiguous sequence of instructions
that transforms a given input into a desired output within a bounded
number of steps~\cite{hoos2005intro}. In the context of optimization,
the input is a problem instance and the output is a candidate solution
together with its objective value; algorithm quality is judged both by
the solutions returned and the computational resources consumed.

\paragraph{Adaptive Algorithms.}

An algorithm is \textit{adaptive} if each action at step $t$ is chosen
as a function of the history of prior queries and their responses,
$(x_1,f(x_1)),\ldots,(x_{t-1},f(x_{t-1}))$; a \textit{non-adaptive}
algorithm such as pure random sampling fixes all query points in
advance and serves as a natural complexity
baseline~\cite{droste2006blackbox}. All algorithms studied in this
project are fully adaptive.

\paragraph{Deterministic and Randomized Algorithms.}
% Definitions and differences.
A \textit{deterministic} algorithm produces the same sequence of
operations and the same output on every execution for a given input.
A \textit{randomized} algorithm, by contrast, makes probabilistic
choices during execution, so its behavior on a fixed input may vary
across runs. In the black-box optimization setting, randomization is
not only a convenience but a necessity: the No-Free-Lunch theorems
imply that no deterministic algorithm can perform well uniformly
across all problem instances (See~\ref{subsubsec:nfl},
\citet{wolpert2002no}), and randomized search heuristics derive
their practical value from their ability to escape local optima
through stochastic perturbation~\cite{doerr2020theory,hoos2005intro}.
The theoretical analysis of such algorithms therefore concerns
expected quantities (expected runtime, expected solution quality)
rather than worst-case deterministic guarantees.

\subsection{The Black-Box Model}
% Formal oracle model.
% Zeroth-order information.
% Information revealed per query.
% - What structural knowledge white-box methods exploit
% - Gradient availability as the dividing line
% - Computational trade-offs: information richness vs. access cost

In the \textit{black-box optimization} model, the algorithm has no access to the
analytical form of the objective function. It interacts with the
objective exclusively
through an oracle that, upon receiving a query $x \in \mathcal{X}$,
returns $f(x)$ (or
  a noisy estimate
thereof)~\cite{droste2006blackbox,doerr2020theory,lengler2019drift}.
The total number of such
interactions constitutes the
\textit{query complexity} of the algorithm, formalized in
Section~\ref{subsubsec:infocomplexity}~\cite{nocedal2006numerical,
droste2006blackbox}.
The black-box model is appropriate whenever the objective arises from
a simulation,
physical experiment, or any process whose internal structure is
inaccessible or too
complex to exploit analytically. In fact, it has been observed that
\acrshort{bbo} methods are sometimes applied due to their ease of use
rather than necessity~\cite{bajaj2021black}.

In contrast, in \textit{white-box optimization}, some information about
the problem instance is made available to
the algorithm (i.e., properties described in
Subsubsection~\ref{subsubsec:order-methods}, Subsection~\ref{subsec:landprop}).

\subsubsection{Taxonomy}

Three models are relevant to this project. In the
\textbf{unrestricted model}, the algorithm has arbitrary memory and
full access to function values, yielding the weakest restriction and
smallest complexity~\cite{droste2006blackbox}. In the \textbf{unary
unbiased model}, variation operators are invariant under permutations
and bitwise complementations of the search
space~\cite{lehre2010black}; both \acrshort{rls} and the
$(1+1)$-\acrshort{ea} fall into this class, making its \acrshort{bbc}
the tightest applicable lower bound for these algorithms.
\acrshort{sa} fits neither model cleanly (it exploits absolute fitness
differences rather than rankings) so its performance is studied empirically.

\subsubsection{Yao's Minimax Principle}
\label{subsubsec:yaomini}

Lower bounds are proved via Yao's minimax
principle~\cite{yao1977probabilistic}. Let $\mathcal{I}$ be the set
of problem instances, $\mathcal{A}$ the set of deterministic
algorithms, and $T(I,A)$ the query cost (number of oracle calls) of
algorithm $A \in \mathcal{A}$ on instance $I \in \mathcal{I}$. For
any distribution $p$ over instances and $q$ over deterministic
algorithms,
\[
  \min_{A \in \mathcal{A}}\,\mathbb{E}_{I \sim p}[T(I,A)]
  \;\leq\;
  \max_{I \in \mathcal{I}}\,\mathbb{E}_{A \sim q}[T(I,A)].
\]
A lower bound follows by exhibiting a distribution over instances on
which every deterministic algorithm is slow. This parallels the
minimax regret framework (Section~\ref{subsubsec:cr}), replacing the
hitting criterion with a trajectory-level measure. The principle
extends to memory-restricted and elitist models via a suitable
superset construction~\cite{doerr2015onemax} (see
Appendix~\ref{app:bbc-extended}).

\subsection{Complexity Measures}
\subsubsection{Classical Complexity: Time and Space}
The classical measures of algorithmic efficiency are \textit{time
complexity} (the number of elementary operations performed as a
function of input size) and \textit{space complexity} (the amount
of memory consumed). Both are typically expressed using asymptotic
notation: $O(\cdot)$, $\Omega(\cdot)$, and $\Theta(\cdot)$ describe
upper, lower, and tight bounds, respectively, on resource usage as
the input size
$n$ grows.

\subsubsection{Information Complexity: Query and Oracle Complexity}
\label{subsubsec:infocomplexity}
% Number of oracle calls as complexity measure.

In \acrshort{bbo}, the internal structure of $f$ is
inaccessible, so the relevant measure of complexity is the number of
oracle queries
required to solve a problem instead of the
number of arithmetic operations. The \textit{query complexity} of an
algorithm $\mathcal{A}$ on a problem class $\mathcal{F}$ is the
number of calls to the function-evaluation oracle needed to identify
a solution meeting the target criterion. The \textit{black-box
complexity} of a problem class is the minimum query complexity over
all algorithms, formalizing the information-theoretic difficulty of
the problem when only objective values are
observable~\cite{droste2006blackbox}. Lower bounds on black-box
complexity establish that no algorithm can solve the problem in fewer
queries, regardless of its computational power; upper bounds are
witnessed by constructive algorithms. This duality mirrors the
classical worst-case analysis of deterministic algorithms but is
applied to the query-response model introduced in
Section~\ref{subsubsec:oraclecomp}.

\subsubsection{Expected vs.\ Worst-Case Analysis}
% Runtime notions in randomized algorithms.

For deterministic algorithms, worst-case analysis over all possible
inputs is the standard framework. For randomized algorithms, and in
particular for the randomized search heuristics studied in this
project, the primary measure is the \textit{expected runtime}: the
mean number of function evaluations until a target solution is first
found, averaged over the internal randomness of the
algorithm~\cite{droste2002analysis,doerr2020theory}. Expected
runtime analysis is complemented by tail bounds (probabilistic
  guarantees that the runtime does not greatly exceed its
expectation) derived via tools such as Markov's inequality and
concentration arguments~\cite{doerr2020theory}. Worst-case analysis
over problem instances, by contrast, motivates the minimax framework
and Yao's minimax principle, which lower-bounds the expected
complexity of any randomized algorithm by exhibiting a hard
distribution over instances~\cite{droste2006blackbox}
(Subsubsection~\ref{subsubsec:yaomini}).

The \textit{pseudo-Boolean} setting considers functions $f :
\{0,1\}^n \to \mathbb{R}$
defined on binary strings of length $n$. Several benchmark functions
are widely used to
study and compare the behavior of randomized search heuristics, each encoding a
different type of difficulty.~\cite{lengler2019drift,droste2006blackbox}

\subsubsection{OneMax}

The \textsc{OneMax} function counts the number of one-bits in a bit string:
\[
  \textsc{OneMax}(x) = \sum_{i=1}^{n} x_i, \qquad x \in \{0,1\}^n.
\]
The unique global maximum is $x^{*} = 1^n$ with $\textsc{OneMax}(1^n) = n$.
\textsc{OneMax} is the canonical baseline benchmark: the landscape is
unimodal and every
improving move increases the function value by exactly one, providing
a clear gradient
for local search methods. The expected runtime of the
$(1+1)$-\acrfull{ea} on \textsc{OneMax} is
$\Theta(n \log n)$.~\cite{droste2002analysis}

\subsubsection{TwoMax}
The \textsc{TwoMax} problem is a simple bimodal benchmark over bit
strings~\cite{oliveto2014runtime}:
\[
  \textsc{TwoMax}(x) = \max\{\textsc{OneMax}(x), n -
  \textsc{OneMax}(x)\}, \qquad x \in \{0,1\}^n.
\]

It consists of two different symmetric branches (i.e.,
\textsc{ZeroMax} and \textsc{OneMax}) and two symmetric global
optima, $0^n$ and $1^n$. This makes \textsc{TwoMax} useful for
studying whether evolutionary algorithms can preserve diversity and
explore symmetric search-space regions rather than converging to only
one optimum.

\subsubsection{LeadingOnes}

The \textsc{LeadingOnes} function counts the length of the maximal
leading prefix of
ones:
\[
  \textsc{LeadingOnes}(x) = \sum_{i=1}^{n} \prod_{j=1}^{i} x_j.
\]
Equivalently, $\textsc{LeadingOnes}(x)$ equals the index of the first
zero bit minus
one, or $n$ if no zero exists. The unique global optimum is again $1^n$. Because
progress requires the leading bit to be set before any subsequent bit
can contribute,
\textsc{LeadingOnes} is significantly harder than \textsc{OneMax}:
the expected runtime
of the $(1+1)$-\acrshort{ea} is $\Theta(n^2)$.~\cite{droste2002analysis}

\subsubsection{Jump}

The \textsc{Jump} function with parameter $k \in \{1, \ldots, n\}$
coincides with
\textsc{OneMax} on bit strings with at most $n - k$ ones and on the
global optimum
$1^n$, but introduces a region of reduced fitness for strings with
between $n-k+1$ and
$n-1$ ones. This creates a ``gap'' of width $k$ that algorithms
relying on single-bit
improvements cannot cross: the set of strings with exactly $n-k$ ones
constitutes a
local optimum of standard local search, and reaching the global
optimum $1^n$ requires
flipping all $k$ remaining zero bits simultaneously. For the
$(1+1)$-\acrshort{ea} with standard
bit mutation at rate $1/n$, the probability of a specific $k$-bit flip is
$\Theta(n^{-k})$, giving an expected runtime of $\Theta(n^k)$ on
$\textsc{Jump}_k$.

Formally, \textsc{Jump}$_{k,n} \colon \{0,1\}^n \to \mathbb{R}$ is defined as
\[
  \textsc{Jump}_{k,n}(x) \;=\;
  \begin{cases}
    k + \displaystyle\sum_{i=1}^{n} x_i & \text{if }
    \displaystyle\sum_{i=1}^{n} x_i \leq n - k
    \text{ or } \displaystyle\sum_{i=1}^{n} x_i = n, \\[8pt]
    n - \displaystyle\sum_{i=1}^{n} x_i & \text{otherwise.}
  \end{cases}
\]
\citet{droste2002analysis} proved that the
expected running
time of the $(1+1)$-\acrshort{ea} on $\textsc{Jump}_{k,n}$ is
$\Theta(n^k + n\log n)$ for
$k \in \{1, 2, \ldots, n\}$, which simplifies to $\Theta(n^k)$ for $k \geq 2$.

\subsubsection{BinaryValue}

The \textsc{\acrfull{bv}} function interprets a bit
string as a binary
integer by weighting bits exponentially by position:
\[
  \textsc{BinVal}(x) = \sum_{i=1}^{n} 2^{i-1}\, x_i.
\]
The global optimum is $1^n$ with value $2^n - 1$. The exponential
weighting means that
the most significant bit contributes more than the combined value of
all less significant
bits, making \textsc{\acrshort{bv}} highly structured.

\subsubsection{Trap}

The \textsc{Trap} function is a piecewise-linear \textit{function of
unitation} formally analyzed for deception
conditions by \citet{deb1993analyzing}. Its value
depends only on the number of ones $u = \sum_{i=1}^n x_i$, not on
their positions, and with slope-change location $z = n - k$ it is
defined as
\[
  \textsc{Trap}_{k}(x) =
  \begin{cases}
    \dfrac{n}{z}(z - u) & \text{if } u \leq z, \\[8pt]
    \dfrac{n}{k}(u - z) & \text{otherwise.}
  \end{cases}
\]
For $u \leq z$ the gradient points toward the all-zeros attractor
$0^n$; only past the slope change does fitness increase toward the
global optimum $1^n$. Every improving move for a hill-climber
therefore moves \emph{away} from the optimum. Deb and Goldberg
establish that the necessary and sufficient condition for full
deception is that all schemata of order $n-1$ favor the local
optimum, which requires $z \geq \lfloor n/2 \rfloor$~\cite{deb1993analyzing}.

\subsubsection{Plateau}

The \textsc{Plateau} function with parameter $k$ is defined as
\textsc{OneMax} everywhere except in a ball of radius $k$ around the
optimum, where fitness is held constant at the second-best value
$n - k$~\cite{antipov2021precise}:
\[
  \textsc{Plateau}_k(x) =
  \begin{cases}
    n - k & \text{if } n - k < \textsc{OneMax}(x) < n, \\
    \textsc{OneMax}(x) & \text{otherwise.}
  \end{cases}
\]
The global optimum is $1^n$ with value $n$. Unlike \textsc{Jump},
which penalizes solutions near the optimum with a \emph{reduced}
fitness, \textsc{Plateau} assigns the plateau region the same
fitness as the boundary level $n - k$, so there are no true local
optima. Instead, the algorithm performs an unguided random walk
across the $\Theta(n^k)$ plateau points, gaining no fitness signal
until it reaches $1^n$ by flipping between one and $k$ zero-bits
simultaneously.

\citet{antipov2021precise} establish that the expected runtime of the
$(1+1)$-\acrshort{ea} on \textsc{Plateau}$_k$ depends only on the
probability of flipping between one and $k$ bits per step, not on
other properties of the mutation operator, and that the optimal
mutation rate under standard bit mutation is approximately $k/(en)$,
a factor of $\Theta(k)$ above the canonical $1/n$
recommendation. The full runtime expression is given in
Appendix~\ref{app:plateau-runtime}.

\subsubsection{Hierarchical If-and-Only-If (HIFF)}

The \textsc{\acrfull{hiff}} function is a benchmark designed to model
\textit{hierarchical building-block
interdependency}: a property absent from earlier \acrfull{ga} test
problems, which treat sub-problems as
separable~\cite{watson1998modeling}. In \acrshort{hiff},
the optimal solution to a block of bits depends on how that block
combines with its neighbors at the next level of the hierarchy,
making sub-problems \textit{non-separable} in a consistent,
multi-level sense.

The fitness of a bitstring $x \in \{0,1\}^n$, where $n$ must be a
power of two, is defined recursively. A block $B$ of length one
always contributes a fitness of one. For any longer block, if all
bits in $B$ are identical (all zeros or all ones), the block
contributes $|B|$ to fitness and its two halves are recursively
evaluated; otherwise only the recursive evaluation of the two halves
contributes. Formally,
\[
  f(B) =
  \begin{cases}
    1 & \text{if } |B| = 1, \\
    |B| + f(B_L) + f(B_R) & \text{if } |B| > 1 \text{ and all bits
    in } B \text{ are equal,} \\
    f(B_L) + f(B_R) & \text{otherwise,}
  \end{cases}
\]
where $B_L$ and $B_R$ denote the left and right halves of $B$.
The global optimum is achieved by the all-zeros or all-ones
bitstring, giving a maximum fitness of $n(\log_2 n + 1)$. Two
global optima exist and are maximally distant from one another,
a structural feature that makes \acrshort{hiff} acutely challenging
for hill-climbing algorithms: a single bit-flip from either optimum
disrupts the top-level block uniformity and causes a sharp fitness
drop, so local search stagnates at intermediate solutions in which
only lower-level blocks are uniform~\cite{watson1998modeling}.

The landscape is highly multimodal. Secondary optima consist of
strings with contiguous blocks of zeros and ones at each
hierarchical level (for example, the first $n/2$ bits all zero and
the remaining $n/2$ all one for a length-$n$ string), and these
local optima are maximally distant from both global optima.
\citet{oates2000triphase} demonstrate empirically that
this structure induces a characteristic tri-modal performance profile
in evolutionary search: progress occurs in discrete phases
corresponding to the successive exploitation of one-bit, two-bit,
and higher-order mutations needed to escape each tier of local
optima. This property makes \acrshort{hiff} a useful diagnostic
benchmark for regret analysis, as the hierarchical barrier structure
should produce a distinctive staircase signature in the cumulative
regret trajectory.

\subsubsection{MaxSAT}
\textsc{\acrfull{maxsat}} is the optimization variant
of the Boolean \acrfull{sat}. A propositional formula
in \acrfull{cnf} is a conjunction of
\textit{clauses}, each of which is a disjunction of \textit{literals}
(a Boolean variable or its negation). Given such a formula with $m$
clauses over $n$ Boolean variables, the goal is to find a truth
assignment that maximizes the number of satisfied clauses. \acrshort{maxsat} is
NP-hard in general~\cite{hutchison_search_2004} and serves as an
important benchmark for combinatorial optimization algorithms.

\paragraph{Petersen Graph 3-Coloring.}
\label{subsec:petersen}
\begin{figure}[H]
  \centering
  \includegraphics[width=0.4\textwidth]{petersen_graph.pdf}
  \caption{From \cite{weisstein_petersen_graph}, depicting a Petersen
  graph, a cubic graph on 10 vertices and 15 edges}
\end{figure}

The \textit{Petersen graph}~\cite{weisstein_petersen_graph} is a
well-known 3-regular graph on ten vertices and fifteen edges whose
chromatic number is exactly three, meaning a valid 3-colouring exists
and no 2-colouring does. This project encodes the 3-colouring problem
as a \acrshort{maxsat} instance over $n = 30$ binary variables, with
one variable per vertex-color pair: variable $x_{v,c}$ is $1$ if
vertex $v$ is assigned color $c \in \{0,1,2\}$. Three families of
soft clauses encode the coloring constraints: ten \textit{at-least-one}
clauses (each vertex must receive at least one color), thirty
\textit{at-most-one} clauses (each vertex must not hold two colors
simultaneously), and forty-five \textit{edge-conflict} clauses (adjacent
vertices must not share a color), giving $m = 85$ clauses in total.
Because the Petersen graph is 3-colourable, all 85 clauses can be
satisfied simultaneously, so the global optimum is $f^* = 85$ and is
genuinely achievable~\cite{hutchison_search_2004}. This is the key
advantage over random \acrshort{maxsat} instances, whose declared optima
are rarely reachable in practice.

\subsubsection{Tunable Landscapes: NK-Landscapes}
\label{subsubsec:nk}

\textit{NK-landscapes}~\cite{kauffman1987towards} are a parametric
family of pseudo-Boolean
functions controlled by two integers: $N$ (the number of variables)
and $K$ (the number
of epistatic interactions per variable, $0 \leq K \leq N-1$). The
fitness of a string
$x \in \{0,1\}^N$ is defined as
\[
  f(x) = \frac{1}{N} \sum_{i=1}^{N} f_i\!\left(x_i,\, x_{\pi(i,1)},
  \ldots,\, x_{\pi(i,K)}\right),
\]
where each local contribution $f_i : \{0,1\}^{K+1} \to [0,1]$ is
drawn independently
and uniformly at random, and $\pi(i, \cdot)$ denotes the $K$
epistatic neighbors of
variable $i$.

When $K = 0$ the contributions are independent, producing a smooth,
additively separable
landscape analogous to \textsc{OneMax}. As $K$ increases toward $N-1$, epistatic
interactions intensify and the landscape becomes increasingly rugged,
with the number of
local optima growing in the limit. NK-landscapes thus provide a tunable model of
ruggedness and are widely used to study algorithmic scalability and
robustness across a
spectrum of problem difficulties.

\subsection{Landscape Properties}
\label{subsec:landprop}

The geometric properties of $f$ determine which tier of analytical
information is both available and useful. Convexity and Lipschitz
smoothness, the properties that underpin first- and second-order
guarantees, are defined in Appendix~\ref{app:landscape-props}.

\paragraph{Ruggedness and Multimodality.}
% - Multiple local optima, deceptive gradients, epistatic interactions
% - Motivates heuristic / stochastic methods: no first-order
% structure to exploit
% - Forward reference to NK-landscapes as a tunable model of ruggedness
A landscape is \textit{rugged} when it contains many local optima
separated by fitness valleys. In such landscapes the gradient (where
it exists) is deceptive: improving moves at the local scale lead away
from the global optimum, and no first-order structure can be
exploited. Ruggedness arises naturally from \textit{epistatic
interactions}, where the contribution of one variable to the
objective depends on the values of others, preventing any separable
decomposition. As $K$ increases in the NK-landscape model
(Section~\ref{subsubsec:nk}), epistasis intensifies and the number
of local optima grows, providing a tunable axis of landscape
difficulty. Ruggedness is the primary motivation for stochastic
search heuristics: randomized perturbation and acceptance of
worsening moves are the principal tools for escaping local optima
that would permanently trap deterministic hill-climbers.

\subsection{Purpose of Randomized Search Heuristics}

Devising tailored, exact algorithms for arbitrary \acrshort{bbo}
problems is, in general,
intractable. Without structural information about the objective
function, there is no
principled basis on which to prefer one search strategy over another,
and even when
some structure is known, the cost of exploiting it may be prohibitive. This
motivates turning to \acrfull{rsh}s and other
metaheuristics\footnote{\textit{Metaheuristics}
  are search techniques that generalise heuristics observed in natural processes
  (e.g.\ \acrshort{ea}s, Particle Swarm
Optimization)~\cite{abdel2018metaheuristic}} as
general-purpose black-box methods.

\subsubsection{No-Free-Lunch Theorems}
\label{subsubsec:nfl}

This motivation is placed on theoretical footing by the
\textit{\acrfull{nfl} theorems}~\cite{wolpert2002no}, which formalize the
impossibility of a universally superior algorithm. The theorems state
that, for any two
black-box optimization algorithms, their performance averaged uniformly over all
possible objective functions on a finite domain is identical; any
gain achieved on one
subset of functions is offset by a corresponding loss on the complementary
subset~\cite{wolpert2002no}. No single algorithm can therefore
outperform all others
across every problem class: the choice of algorithm implicitly
encodes assumptions about
problem structure, and those assumptions can always be violated.

For the pseudo-Boolean domain $\{0,1\}^n$, any \acrshort{rsh} that
performs strongly
on benchmarks such as \textsc{OneMax} or \textsc{LeadingOnes} must
underperform on some
complementary class of functions. Complexity results such as
$\mathbb{E}[\tau_{f^*}] = \Theta(n \log n)$ for the $(1+1)$-\acrshort{ea} on
\textsc{OneMax}~\cite{droste2002analysis} are meaningful purely because
\textsc{OneMax} is a natural representative of unimodal functions.
The \acrshort{nfl}
theorems therefore make conditioning comparisons on a structured
problem class a theoretical necessity not a
practical convenience, which motivates the benchmark suite
defined in Section~\ref{subsec:bbo-perf}.

\subsection{Randomized Search Heuristic Algorithms}

\subsubsection[\texorpdfstring{Evolutionary Algorithms
(EA)}{Evolutionary Algorithms (EA)}]{Evolutionary Algorithms (\acrshort{ea})}
\label{subsubsec:ea}
\begin{wrapfigure}{i}{0.7\textwidth}
  \includegraphics[width=0.7\textwidth]{metaheuristics_taxonomy.pdf}
  \caption[Taxonomy of Metaheuristics based on source of
  inspiration]{Taxonomy of Metaheuristics based on source of
    inspiration. \acrshort{ea}s and \acrshort{sa} are examples. Adopted
  from \cite[Figure 5]{rajwar2023exhaustive}}
  \label{fig:mettax}
\end{wrapfigure}

Evolutionary algorithms are a broad family of population-based,
randomized search
heuristics inspired by biological evolution. It belongs to the family
of \textit{metaphor-based} metaheuristics, as shown in
Figure~\ref{fig:mettax}. In discrete
optimization, the canonical
paradigm maintains one or more \emph{candidate solutions} (bit
strings) and iterates
two operators: \emph{mutation}, which perturbs a solution stochastically, and
\emph{selection}, which retains solutions according to their objective
value~\cite{droste2002analysis,doerr2020theory}. The simplest instance of this
paradigm, the $(1+1)$-EA, serves as the theoretical baseline
throughout this project.

\paragraph{The $(1+1)$-EA.}
\label{par:oneone-ea}
The $(1+1)$-EA maintains a single current solution $x \in \{0,1\}^n$. In each
iteration it produces one offspring $y$ by \emph{standard bit
mutation}: every bit is
flipped independently with probability $p$. The offspring replaces
the parent if and
only if $f(y) \geq f(x)$ (\emph{elitist selection}). With the
standard choice $p =
1/n$ the expected number of flipped bits is exactly one. The
resulting process is a
homogeneous Markov chain on $\{0,1\}^n$ with a non-decreasing fitness
sequence~\cite{droste2002analysis,witt2013tight}.

\begin{algorithm}[H]
  \caption{The $(1+1)$-EA with mutation probability $p \in (0,1)$}
  \label{alg:oneone-ea}
  \begin{algorithmic}[1]
    \State Sample $x \in \{0,1\}^n$ uniformly at random; evaluate $f(x)$
    \Repeat
    \State $y \gets x$
    \For{$i = 1, \ldots, n$}
    \State flip $y_i$ with probability $p$, independently
    \EndFor
    \If{$f(y) \geq f(x)$} $x \gets y$ \EndIf
    \Until{termination criterion met}
  \end{algorithmic}
\end{algorithm}

\noindent The classical runtime results for $p = 1/n$ are
\[
  \mathbb{E}[T_{\textsc{OneMax}}] = e\,n\ln n + O(n)
  \qquad \text{and} \qquad
  \mathbb{E}[T_{\textsc{LeadingOnes}}] = \tfrac{e-1}{2}\,n^2 + o(n^2),
\]
and the $\Theta(n\log n)$ bound extends to \emph{all} linear
pseudo-Boolean functions
with non-zero weights~\cite{witt2013tight}. On general landscapes the
algorithm may require
exponential time: even simple quadratic and unimodal functions can trap it
indefinitely without restarts or adaptive
mutation~\cite{droste2002analysis}. Advanced variants and
implementation-aware refinements are discussed in
Appendix~\ref{app:ea}.

\subsubsection{Simulated Annealing}
\acrfull{sa} is a randomized local-search method that differs from a
pure hill-climber by sometimes accepting worse neighboring solutions.
At each iteration, the algorithm proposes a neighboring candidate
solution and compares its objective value with that of the current
solution. Improving moves are accepted immediately. Worsening moves may
also be accepted, but only with a probability controlled by a
temperature parameter. High temperatures make such moves relatively
likely, encouraging exploration; low temperatures make them rare,
causing the search to behave increasingly like local search.

\begin{algorithm}[H]
  \caption{Simulated Annealing with cooling schedule $T(\cdot)$}
  \label{alg:sa}
  \begin{algorithmic}[1]
    \State Sample $x \in \{0,1\}^n$ uniformly at random; evaluate $f(x)$
    \State $t \gets 1$;\; $x^{*} \gets x$;\; $f^{*} \gets f(x)$
    \Repeat
    \State $y \gets x$;\; flip $y_i$ for a uniformly random $i \in
    \{1,\ldots,n\}$
    \State evaluate $f(y)$;\; $\delta \gets f(y) - f(x)$
    \State $T \gets \max\!\bigl(T_{\min},\; T(t)\bigr)$
    \Comment{clamp to avoid $T=0$}
    \State $t \gets t + 1$
    \If{$\delta \geq 0$ \textbf{or} $\mathrm{Uniform}(0,1) < \exp(\delta / T)$}
    \State $x \gets y$;\; $f(x) \gets f(y)$ \Comment{Metropolis acceptance}
    \EndIf
    \If{$f(x) > f^{*}$} $x^{*} \gets x$;\; $f^{*} \gets f(x)$ \EndIf
    \Until{termination criterion met}
    \State \Return $x^{*}$
  \end{algorithmic}
\end{algorithm}

This acceptance rule is a special case of the
Metropolis--Hastings acceptance
criterion~\cite{hastings1970monte,metropolis1953equation}. In
general, a proposed move
is accepted according to the ratio between the probability assigned to
the proposed state and the probability assigned to the current state.
For the symmetric neighborhood proposals used in this project, the
proposal probabilities cancel, leaving only the ratio of the
temperature-dependent state weights. In the maximization convention
used throughout this project, a candidate solution with objective value
\(f(x)\) is assigned weight
\[
  \exp(f(x)/T)\footnote{The classical physics notation writes the
    Boltzmann factor as
    \(\exp(-E/T)\), where \(E\) is an energy to be minimized. This project
    uses a maximization objective \(f\), so the equivalent form is
    \(\exp(f/T)\). The two conventions agree under the identification
  \(f=-E\).\label{fn:bolmaxinot}},
\]
where \(T>0\) is the current temperature. Hence, a proposed move from
\(x\) to \(x'\) is accepted automatically when \(f(x') \geq f(x)\), and
is accepted with probability
\[
  \exp((f(x') - f(x))/T)
  \quad \text{when } f(x') < f(x).
\]
Thus, small decreases in objective value are more likely to be accepted
than large decreases, and all worsening moves become less likely as the
temperature decreases.

\begin{figure}[H]
  \centering
  \includegraphics[width=0.6\textwidth]{boltzmann_dist.png}
  \caption{From \cite[Figure 1]{nityananda2012bose}: The Boltzmann
    distribution of particles over energy states.
    The horizontal axis represents
    the energy \(E\) of a physical state; in simulated annealing this
    corresponds to the cost or energy \(E(x)\) of a candidate solution
    \(x\), where lower values are preferred (minimization, see
    footnote ~\ref{fn:bolmaxinot}). The vertical axis represents
    the number of particles occupying states of that energy; in simulated
    annealing this corresponds to the relative probability assigned to
    candidate solutions with that energy. At high temperature, the
    distribution is flatter, so higher-energy, worse solutions still have
    non-negligible probability and the algorithm explores more freely. As
    the temperature decreases, probability mass concentrates on
    lower-energy solutions, making worsening moves increasingly unlikely
    and causing the search to behave more like local descent.
  }
  \label{fig:boltzmann-sa}
\end{figure}

This places \acrshort{sa} in direct relation to the Gibbs distribution,
also called the Boltzmann distribution in its statistical-mechanical
setting. The distribution assigns higher probability to states with
larger objective value at a fixed temperature, while the cooling
schedule gradually concentrates probability mass on high-quality
solutions~\cite{kirkpatrick1983optimization,bertsimas1993simulated}.
The practical role of \acrshort{sa} is therefore to use this
temperature-dependent acceptance mechanism to balance exploration early
in the run with exploitation later in the run. Formal convergence
conditions for cooling schedules are given in Appendix~\ref{app:hajek}.

% \acrfull{sa} is a probabilistic local search
% algorithm that extends iterative improvement by occasionally accepting
% worsening moves, allowing escape from local optima that would otherwise
% trap a pure hill-climber. The method was introduced by
% \citet{kirkpatrick1983optimization}, who drew an
% explicit analogy with the physical process of annealing in solids:
% a material is first melted at high temperature, then cooled slowly
% so that atoms settle into a stable, low-energy crystalline configuration.

% In the optimization context, the algorithm maintains a current solution
% $x \in \mathcal{X}$ and at each step proposes a random neighbor
% $y \in \mathcal{N}(x)$. If $f(y) \geq f(x)$, the move is accepted
% unconditionally. If $f(y) < f(x)$, the move is accepted with
% probability
% \[
%   \exp\!\left(\frac{f(y) - f(x)}{T(t)}\right),
% \]
% where $T(t) > 0$ is the \textit{temperature} at step $t$, a control
% parameter that decreases over time according to a \textit{cooling
% schedule}~\cite{kirkpatrick1983optimization,bertsimas1993simulated}.
% At high temperatures the acceptance criterion is nearly uniform,
% permitting broad exploration; as $T(t) \to 0$ the algorithm collapses
% toward a greedy hill-climber that accepts only improvements.

% The theoretical behavior of \acrshort{sa} is governed by the \textit{cooling
% schedule}. Modelling the process as a time-inhomogeneous Markov chain,
% Hajek~\cite{hajek1988cooling} established a sharp necessary and
% sufficient condition on the schedule for convergence in probability to
% the global optimum; the condition relates the decay rate of $T(t)$ to
% the depth $d^{*}$ of the deepest non-global local minimum. The full
% statement and its implications are given in
% Appendix~\ref{app:hajek}. In practice, the schedule satisfying this
% condition cools logarithmically, which is theoretically optimal but
% often too slow; faster schedules reduce \acrshort{sa} to a stochastic
% hill-climber once the temperature reaches zero. The choice of cooling
% schedule therefore encodes a fundamental trade-off between solution
% quality and computational budget, and is the primary design decision
% when applying \acrshort{sa} to any combinatorial
% problem~\cite{kirkpatrick1983optimization,hajek1988cooling}.

\paragraph{Cooling Schedules.}

Three cooling schedules were evaluated in this project.
\subparagraph{Logarithmic cooling}
It decreases very slowly and is the only schedule considered
here with a classical asymptotic
convergence guarantee: for schedules of this form, convergence is
obtained when $d \geq d^{*}$, $d^{*}$ the depth (or height) of the
largest local optima~\cite{hajek1988cooling}.

\begin{algorithm}[H]
  \caption{Logarithmic cooling schedule}
  \label{alg:cool-log}
  \begin{algorithmic}[1]
    \Require scaling constant $d > 0$, evaluation index $t \geq 1$
    \State \Return $T(t) = \dfrac{d}{\ln(t + 1)}$
    \Comment{$d \geq d^{*}$ guarantees asymptotic
    convergence~\cite{hajek1988cooling}}
  \end{algorithmic}
\end{algorithm}

\subparagraph{Exponential cooling}

It follows the geometric pattern used in the original
simulated annealing experiments~\cite{kirkpatrick1983optimization}
and therefore reaches very low temperatures quickly under a finite
evaluation budget.
\begin{algorithm}[H]
  \caption{Exponential cooling schedule}
  \label{alg:cool-exp}
  \begin{algorithmic}[1]
    \Require initial temperature $T_0 > 0$, decay factor $\alpha \in (0,1)$,
    evaluation index $t \geq 0$
    \State \Return $T(t) = T_0 \cdot \alpha^{\,t}$
  \end{algorithmic}
\end{algorithm}

\subparagraph{Linear cooling}

It decreases uniformly to a prescribed floor and is used here as a
practical fixed-horizon baseline in the experiments; unlike
logarithmic cooling, it carries no classical asymptotic convergence
guarantee in this setting~\cite{bertsimas1993simulated}.
\begin{algorithm}[H]
  \caption{Linear cooling schedule}
  \label{alg:cool-lin}
  \begin{algorithmic}[1]
    \Require $T_0 > T_f \geq 0$, budget $t_{\max}$, evaluation index $t \geq 0$
    \State \Return $T(t) = \max\!\Bigl(T_f,\; T_0 - (T_0 -
    T_f)\,\tfrac{t}{t_{\max}}\Bigr)$
  \end{algorithmic}
\end{algorithm}

Suite~11~(Subsection~\ref{subsec:suit11})
compares the regret profiles of these schedules, focusing on when
temperature collapse occurs and when instantaneous regret
stabilizes.

\subsubsection[\texorpdfstring{Randomized Local Search
(RLS)}{Randomized Local Search (RLS)}]{\acrfull{rls}}

\acrfull{rls} is a single-solution local search heuristic that at
each step selects a neighbor of the current solution uniformly at
random and moves to it if the objective value does not decrease
(\textit{elitist selection}). In the pseudo-Boolean setting, the
standard neighborhood is the Hamming-1 ball: exactly one bit is
chosen uniformly at random and flipped (`local`).~\cite{hoos2005sls}

Three variants are studied in this project: greedy \acrshort{rls},
$\varepsilon$-greedy \acrshort{rls}, and decaying
$\varepsilon$-greedy \acrshort{rls}.

\paragraph{Stochastic Hill-Climber -
Greedy \acrshort{rls}}

The \textit{greedy} \acrshort{rls} variant accepts only improving or
equal-quality moves, making it a pure hill-climber with no mechanism
to escape local optima.

\begin{algorithm}[H]
  \caption{Greedy RLS (Stochastic Hill Climber)}
  \label{alg:rls-greedy}
  \begin{algorithmic}[1]
    \State Sample $x \in \{0,1\}^n$ uniformly at random; evaluate $f(x)$
    \State $t \gets 1$;\; $x^{*} \gets x$;\; $f^{*} \gets f(x)$
    \Repeat
    \State $y \gets x$;\; flip $y_i$ for a uniformly random $i \in
    \{1,\ldots,n\}$
    \State evaluate $f(y)$;\; $t \gets t + 1$
    \If{$f(y) \geq f(x)$} $x \gets y$;\; $f(x) \gets f(y)$ \EndIf
    \If{$f(x) > f^{*}$} $x^{*} \gets x$;\; $f^{*} \gets f(x)$ \EndIf
    \Until{termination criterion met}
    \State \Return $x^{*}$
  \end{algorithmic}
\end{algorithm}

\paragraph[\texorpdfstring{$\varepsilon$-greedy RLS}{epsilon-greedy
RLS}]{$\varepsilon$-greedy \acrshort{rls}}

The \textit{$\varepsilon$-greedy} variant accepts a random Hamming-1
neighbor unconditionally with probability $\varepsilon$ and otherwise
applies elitist selection. This introduces a controlled exploration
rate while still favoring improving moves most of the time.

\begin{algorithm}[H]
  \caption{$\varepsilon$-Greedy RLS}
  \label{alg:rls-eps}
  \begin{algorithmic}[1]
    \Require exploration rate $\varepsilon \in (0,1)$
    \State Sample $x \in \{0,1\}^n$ uniformly at random; evaluate $f(x)$
    \State $t \gets 1$;\; $x^{*} \gets x$;\; $f^{*} \gets f(x)$
    \Repeat
    \State $y \gets x$;\; flip $y_i$ for a uniformly random $i \in
    \{1,\ldots,n\}$
    \State evaluate $f(y)$;\; $t \gets t + 1$
    \If{$\mathrm{Uniform}(0,1) < \varepsilon$}
    \State $x \gets y$;\; $f(x) \gets f(y)$ \Comment{explore unconditionally}
    \ElsIf{$f(y) \geq f(x)$}
    \State $x \gets y$;\; $f(x) \gets f(y)$ \Comment{exploit}
    \EndIf
    \If{$f(x) > f^{*}$} $x^{*} \gets x$;\; $f^{*} \gets f(x)$ \EndIf
    \Until{termination criterion met}
    \State \Return $x^{*}$
  \end{algorithmic}
\end{algorithm}

\paragraph[\texorpdfstring{Decaying $\varepsilon$-greedy RLS}{Decaying
epsilon-greedy RLS}]{Decaying $\varepsilon$-greedy \acrshort{rls}}

The \textit{decaying $\varepsilon$-greedy} variant reduces the
exploration probability over time according to
$\varepsilon_t = \varepsilon_0 / t$. This concentrates exploration
early in the search budget and gradually converges toward greedy
behavior as the run progresses.

\begin{algorithm}[H]
  \caption{Decaying-$\varepsilon$-Greedy RLS}
  \label{alg:rls-decay}
  \begin{algorithmic}[1]
    \Require initial exploration rate $\varepsilon_0 \in (0,1)$
    \State Sample $x \in \{0,1\}^n$ uniformly at random; evaluate $f(x)$
    \State $t \gets 1$;\; $x^{*} \gets x$;\; $f^{*} \gets f(x)$
    \Repeat
    \State $y \gets x$;\; flip $y_i$ for a uniformly random $i \in
    \{1,\ldots,n\}$
    \State evaluate $f(y)$;\; $\varepsilon_t \gets \varepsilon_0 /
    t$;\; $t \gets t + 1$
    \If{$\mathrm{Uniform}(0,1) < \varepsilon_t$}
    \State $x \gets y$;\; $f(x) \gets f(y)$ \Comment{explore
    (probability decays as $1/t$)}
    \ElsIf{$f(y) \geq f(x)$}
    \State $x \gets y$;\; $f(x) \gets f(y)$ \Comment{exploit}
    \EndIf
    \If{$f(x) > f^{*}$} $x^{*} \gets x$;\; $f^{*} \gets f(x)$ \EndIf
    \Until{termination criterion met}
    \State \Return $x^{*}$
  \end{algorithmic}
\end{algorithm}

\subsection{Performance Measures in BBO}
\label{subsec:bbo-perf}
% - Expected runtime (hitting time to reach the optimum or a quality threshold)
%   as the standard measure in EA theory  [droste2002analysis; doerr2020theory]
% - Fixed-budget analysis: quality of best-found solution after T queries
%   [Jansen & Zarges 2014 or similar]
% - Fixed-target analysis: minimum budget to reach a target quality with
%   probability >= 1-delta  [!CITE!]
% - Limitations of these measures:
%     Binary in nature (found / not found)
%     Ignore the trajectory of quality improvement
%     Do not penalise "wasteful" queries on already-known-poor regions
% - This gap motivates regret as a more informative measure
The theoretical analysis of \acrshort{bbo} algorithms rests on three
complementary frameworks, each fixing one dimension of the problem and
analyzing what can be guaranteed along the other.

\paragraph{Expected runtime.}
% - The canonical measure: E[tau_{f*}], expected evaluations to first
% reach the optimum
% - Equivalent to the expected first hitting time of the target level f*
% - Central to EA theory: Theta(n log n) for (1+1) EA on OneMax,
%   Theta(n^2) for LeadingOnes, Theta(n^k) for Jump_k  [droste2002analysis]
% - Analysis via drift theory, Markov chains, the fitness-level method
%   [doerr2020theory; lengler2019drift]
% - Well-suited for establishing complexity hierarchies and ranking algorithms
%   on a given benchmark

The canonical measure is the \textit{expected optimization time}:
the expected number of evaluations until an optimal solution is first
queried~\cite{droste2002analysis,doerr2020theory}. Formally, the
\textit{first hitting time} of the global optimum is
\[
  \tau_{f^*} \;:=\; \min\bigl\{t \in \mathbb{N} : f(x_t) = f^*\bigr\},
\]
and performance is summarized by $\mathbb{E}[\tau_{f^*}]$. Canonical results are
$\Theta(n \log n)$ for the $(1+1)$-\acrshort{ea} on \textsc{OneMax},
$\Theta(n^2)$ on \textsc{LeadingOnes}, and $\Theta(n^k)$ on
$\textsc{Jump}_k$~\cite{droste2002analysis}, derived via drift
theory~\cite{lengler2019drift} and Markov chain
arguments~\cite{doerr2020theory}.

\paragraph{Fixed-budget analysis.}
\label{subsubsec:fixed-budget}
% - Orthogonal view: fix T, characterize E[f*_T(A)] = expected
% quality within budget
%   [jansen2014fixed]
% - Practically motivated: practitioner cannot in general detect optimality;
%   the run terminates at budget exhaustion regardless
% - Different ranking than expected runtime is possible:
%   an algorithm with larger E[tau_{f*}] may still dominate within a
% given budget
% - Connects directly to simple regret: minimising simple regret <=>
% maximising E[f*_T]
% - Forward ref: this is the fixed-budget view of performance;
%   §\ref{sec:regret} generalises it to the entire trajectory

In \textit{fixed-budget analysis}~\cite{jansen2014fixed} the budget $T$ is fixed
and the quantity of interest is $\mathbb{E}[f^{*}_T(\mathcal{A})]$, the expected
quality of the best solution found within $T$ evaluations. This is practically
motivated: a user typically cannot detect optimality and the run terminates at
budget exhaustion regardless. Budget-based rankings can reverse those from
$\mathbb{E}[\tau_{f^*}]$, and maximizing $\mathbb{E}[f^{*}_T]$ is equivalent
to minimizing expected simple regret at budget $T$ (Section~\ref{subsubsec:sr}).

\paragraph{Fixed-Target Analysis and Runtime Profiles.}
\label{subsubsec:fixed-target}
% - Fixed-target: for a quality threshold v, characterise the
% distribution of tau_v
%   [Doerr & Lengler 2015 - OneMax in black-box models with restrictions]
% - More informative than the single tau_{f*}: a full collection of
% hitting times
%   across all levels v gives the expectation-based runtime profile
% [pinto2018practice]
% - Distributional profile P(v,t) = P(tau_v > t): the primitive
% object from which
%   E[tau_v] and subsequently cumulative regret are derived
%   (full formal treatment in Chapter X / Runtime Profiles section)
% - Practical value: identifies crossover levels where one algorithm overtakes
%   another, guiding adaptive strategy design  [pinto2018practice]
% - Pinto & Doerr's open question: a combined anytime measure is needed

\textit{Fixed-target analysis} extends the single hitting time to every
intermediate fitness level $v$, characterizing the distribution of
\[
  \tau_v \;:=\; \min\bigl\{t \in \mathbb{N} : f(x_t) \geq v\bigr\}.
\]
\citet{pinto2018practice} show that reporting the full
collection $(\mathbb{E}[\tau_v])_v$ reveals crossover fitness levels where
one algorithm overtakes another, behavior invisible to $\mathbb{E}[\tau_{f^*}]$
alone. Chapter~\ref{cha:profiles} develops this further.

\paragraph{Shared limitation.}
\label{subsubsec:perf-limitations}
% - Expected runtime is binary: found / not found at a single threshold
% - Fixed-budget gives one scalar per budget T, ignores the trajectory
% - Even runtime profiles are a collection of per-level scalars, not a single
%   summary of the entire quality trajectory
% - None of these penalise "wasted" queries on suboptimal regions:
%   an algorithm that stagnates at low fitness and then jumps to the optimum
%   is indistinguishable from one that makes steady progress, by
% E[tau_{f*}] alone
% - Pinto & Doerr explicitly call for a combined anytime measure from ML/online
%   learning  [pinto2018practice, p.22]; regret is the natural candidate
% - Transition: §2.5 (Sequential Decision-Making) and §2.6 (Regret Formalism)
%   develop this measure from first principles; §2.7 brings it back to BBO

All three frameworks measure performance at isolated thresholds rather than
over the entire quality trajectory, and none penalizes an algorithm for spending
many evaluations at low fitness. A strategy that stagnates and then finds the
optimum in a single lucky step is indistinguishable from one making steady
progress under $\mathbb{E}[\tau_{f^*}]$ alone.
\citet{pinto2018practice} call explicitly for \textit{combined anytime
measures} from machine learning, naming regret as a natural candidate while
noting that a satisfying formulation remains open.

\section{Sequential and Online Optimization Frameworks}
\label{sec:online}
The regret framework adopted in this project originates in the
online learning and sequential decision-making literature. This
section introduces the
minimal concepts from that tradition, the exploration--exploitation
trade-off, the multi-armed bandit model, and Bayesian optimization,
that are required to make the regret vocabulary of
Section~\ref{sec:regret} rigorous and to situate this project
within the broader landscape of black-box optimization research.

\subsection{Exploration-Exploitation Trade-off}
%   Gathering information (exploration) vs. using current best
%   knowledge (exploitation)
% - Why pure exploitation fails (local optima / suboptimal arms)
% - Why pure exploration fails (never converges to good solutions)
% - This tension is shared with BBO

Sequential black-box optimization is governed by the
exploration--exploitation trade-off. At each round, the algorithm
must either explore uncertain regions of the search space to gain
information or exploit the currently best-known region to obtain good
values
immediately~\cite{cesa2006prediction,bubeck2012regret,lattimore2020bandit}
Since each evaluation is both a measurement and a search step,
neither extreme is satisfactory: pure exploitation can lock onto a
suboptimal region, while pure exploration wastes budget.

This tension is central to regret-based analysis. Regret measures the
cost of not knowing the objective in advance, so it captures both
slow identification of good regions and repeated evaluation of poor ones.

\subsection{Multi-Armed Bandits}
% - Formal stochastic bandit model: K arms, reward distributions, T rounds
% - Objective: maximize cumulative reward (equivalently, minimize regret)
% - Stochastic MAB: UCB1, Thompson Sampling - brief intuition  [!CITE!]
% - Contextual bandits: features available at each round  [!CITE!]
% - Pure exploration / best-arm identification: goal is simple regret
%   at the end of the horizon, not cumulative regret  [bubeck2011pure]
% - Connection to BBO: each candidate solution is an "arm"; the oracle
%   is the reward; the algorithm decides which arm to pull
The \acrfull{mab} model is the basic framework for sequential
decision-making under partial
feedback~\cite{bubeck2012regret,lattimore2020bandit}. In the
stochastic $K$-armed setting, an algorithm chooses an arm $A_t \in
\{1,\dots,K\}$ at each round and observes only the reward of that
arm. If arm $i$ has mean reward $\mu_i$ and $\mu^*=\max_i \mu_i$, the
standard pseudo-regret is
\[
  \bar{R}_T = T\mu^* - \sum_{t=1}^{T} \mu_{A_t}
  = \sum_{i:\Delta_i>0} \Delta_i \, \mathbb{E}[N_i(T)],
\]
where $\Delta_i=\mu^*-\mu_i$ is the sub-optimality gap and $N_i(T)$ is
the number of pulls of arm $i$ up to time
$T$~\cite{bubeck2012regret,lattimore2020bandit}. Regret is therefore
driven by how often the learner samples suboptimal actions.

\subsection{Bayesian Optimization}
% - Surrogate-model approach to expensive BBO  [!CITE Mockus / Shahriari!]
% - Gaussian process surrogate + acquisition function (EI, UCB, PI)
% - Regret in BO: cumulative and simple regret bounds for GP-UCB
%   [Srinivas et al. 2010 - key reference]
% - BO as the bridge domain: it lives at the intersection of online learning
%   and BBO, and has the most mature regret theory outside the bandit literature
% - Distinction from the RSH setting: BO uses a learned surrogate;
%   RSHs are memoryless or use no probabilistic model

The closest existing body of work applying regret to black-box
optimization is \acrfull{bo}~\cite{jones1998efficient,
shahriari2015taking, frazier2018tutorial}. \acrshort{bo} treats the optimization
as a sequential learning problem: a probabilistic surrogate
(typically a Gaussian process) models the unknown objective, and an
acquisition function selects the next query by trading predicted
value against uncertainty. Incidentally, it has yielded a mature regret
theory: the GP-UCB algorithm admits sublinear cumulative-regret
guarantees under Gaussian-process or
RKHS\footnote{\textit{abbr.}~Reproducing kernel Hilbert space.
(Optionally, see \cite{ghojogh2021reproducing})} assumptions
\cite{srinivas2009gaussian, chowdhury2017kernelized, scarlett2017lower}.
These guarantees rely on surrogate assumptions
that are natural in low-dimensional continuous spaces but do not
transfer directly to the pseudo-Boolean domains $\{0,1\}^n$ studied
here, where kernel engineering is required and acquisition
optimization is itself an NP-hard combinatorial
problem~\cite{shahriari2015taking}. This motivates studying regret
for the simpler, surrogate-free randomized search heuristics
that form the subject of this project.
\section{Computational Regret}
\label{sec:regret}
Having established both the black-box setting and the online
learning framework from which regret originates, this section
defines regret formally and identifies the specific variant,
incumbent \acrfull{cr}, that is used as the primary
performance measure throughout the experimental analysis in
Chapter~\ref{cha:exper}.

\subsection{Definition of Regret}
% Formal mathematical definition.
% Regret vs. loss.
Let $f : \mathcal{X} \to \mathbb{R}$ be an objective function with
global optimum value
\\$f^{*} = \max_{x \in \mathcal{X}} f(x)$. An algorithm that queries points
$x_1, x_2, \ldots, x_T$ incurs \textit{regret} by querying suboptimal
solutions. Regret
quantifies the gap between the algorithm's performance and that of an
oracle always
querying the optimum~\cite{cesa2006prediction}.

\subsection{Types of Regret}

\subsubsection{Instantaneous Regret}
% Per-round performance gap.
The \textit{instantaneous regret} at round $t$ is
\[
  \rho_t = f^{*} - f(x_t),
\]
capturing the suboptimality of the single query made at step $t$.
A related variant tracks the \textit{incumbent} (best-so-far)
solution rather than
the current query:
\[
  \rho^{\mathrm{inc}}_t = f^{*} - f(\hat{x}_t),
  \quad \hat{x}_t = \operatorname{arg\,max}_{s \leq t} f(x_s).
\]
Because $\hat{x}_t$ is non-decreasing in quality, $\rho^{\mathrm{inc}}_t$ is
monotonically non-increasing over $t$, and is equivalent to simple
regret evaluated at time $t$(~\ref{subsubsec:sr}).

\subsubsection{Cumulative Regret}
\label{subsubsec:cr}
% Definition, interpretation, and relationship to IR.
Two \acrfull{cr} variants arise naturally and serve different purposes.

The \textit{\acrfull{cr}} after $T$ rounds is~\cite{cesa2006prediction}
\[
  R_T = \sum_{t=1}^{T} \rho_t = \sum_{t=1}^{T} \bigl(f^{*} - f(x_t)\bigr).
\]
This is the standard formulation from the online learning literature:
it sums the
instantaneous sub-optimality of each individual query, regardless of
what the algorithm
has previously observed. Minimizing $R_T$ corresponds to performing well
\textit{throughout} the optimization process, penalizing every
suboptimal query equally.

The \textit{incumbent \acrshort{cr}} instead tracks the best-so-far process
\[
  B(t) \;:=\; \max_{s \leq t} f(x_s),
\]
yielding
\[
  R^{\mathrm{inc}}_T = \sum_{t=1}^{T} \rho^{\mathrm{inc}}_t = \sum_{t=1}^{T}
  \bigl(f^{*} - B(t)\bigr).
\]
Since $B(t)$ is non-decreasing in $t$, the per-step contributions
$\rho^{\mathrm{inc}}_t$ are non-increasing, and $R^{\mathrm{inc}}_T
\leq R_T$ for all
$T$. The incumbent formulation is the appropriate notion when the
practitioner retains
the best solution found rather than only the most recent query, which
is the standard
convention for the randomized search heuristics studied in this
project. All regret
analyses in Chapter~\ref{cha:exper} use $R^{\mathrm{inc}}_T$ unless
otherwise stated.

In continuous time, the incumbent \acrshort{cr} is approximated
via a left-hold
(piecewise-constant) integral
\[
  R^{\mathrm{inc}}_T \approx \sum_{t=1}^{T}
  \rho^{\mathrm{inc}}_{t-1}\,\Delta t_{\,t},
  \quad \Delta t_{\,t} = t - t_{t-1},
\]
which accounts for unequal spacing between evaluations when function
evaluation time is
non-uniform. In all experiments reported here, evaluations are
unit-cost and time is
measured in discrete query counts, so $\Delta t_t = 1$ throughout and
this reduces to
the standard discrete sum.

\subsubsection{Simple Regret}
% Final recommendation quality.
\label{subsubsec:sr}
The \textit{\acrfull{sr}} after $T$ rounds is~\cite{bubeck2011pure}
\[
  r_T = f^{*} - f(\hat{x}_T),
\]
where $\hat{x}_T = \operatorname{arg\,max}_{t \leq T} f(x_t)$ is the
best solution
encountered. Simple regret measures the quality of the \textit{final
output} and is the
appropriate notion when the goal is to return a good solution rather
than to perform well
during the search. Note that $r_T = \rho^{\mathrm{inc}}_T$: simple
regret is the incumbent instantaneous regret at the final round.

When comparing algorithms across multiple independent runs, the
standard scalar summary
is the \textit{expected simple regret}
\[
  \mathbb{E}[r_T] = \frac{1}{N}\sum_{i=1}^{N} r_T^{(i)},
\]
where $r_T^{(i)}$ denotes the simple regret of the $i$-th run.

\subsubsection{Normalized Regret}
% Scale-free comparison across problem instances.
To enable comparison across problem instances with different
objective scales, simple
regret can be normalized by the range of attainable fitness values:
\[
  \tilde{r}_T = \frac{f^{*} - f(\hat{x}_T)}{f^{*} -
  f_{\mathrm{worst}}} \in [0, 1],
\]
where $f_{\mathrm{worst}}$ is the worst observed or theoretical
fitness value. A value of
$0$ indicates an optimal solution was found; a value of $1$ indicates
no improvement over
the worst case. When $f^{*} = f_{\mathrm{worst}}$ (a degenerate
constant landscape), the
problem is trivially solved and normalized regret is defined as $0$.

\subsection{Regret Bounds}
% - Upper bounds (algorithmic): what a given algorithm guarantees
%     UCB1: O(sqrt(KT log T)) cumulative regret  [!CITE!]
%     GP-UCB: sublinear cumulative regret for continuous domains
% [Srinivas et al.]
% - Lower bounds (information-theoretic): what no algorithm can beat
%     Omega(sqrt(KT)) for K-armed stochastic bandits
% - Minimax regret: min_{algorithm} max_{instance} R_T
%     Connection to Yao's principle  (already introduced in §2.2.2)
% - Instance-dependent (gap-based) bounds: tighter bounds when suboptimality
%     gaps Delta_i = f^* - f(arm_i) are large  [!CITE!]
% - Sublinear regret as the benchmark: R_T = o(T) implies average per-round
%     regret -> 0, i.e., the algorithm learns to behave near-optimally

The central benchmark of regret analysis is \textit{sublinear regret}: if
$R_T = o(T)$, then the average per-round regret $R_T/T \to 0$, meaning the
algorithm asymptotically performs as well as an oracle with full
information~\cite{cesa2006prediction,bubeck2012regret,lattimore2020bandit}.
Classic results in finite stochastic bandits establish logarithmic gap-dependent
upper bounds and matching lower bounds, showing that logarithmic regret is
unavoidable up to constants under standard
assumptions~\cite{auer2002finite,lai1985adaptive}. In Gaussian-process bandits,
upper and lower bounds depend additionally on the information-gain structure of
the kernel~\cite{srinivas2009gaussian,scarlett2017lower}. These results are
important here primarily as a template: regret provides a well-grounded
complexity measure for sequential optimization, and sublinear regret is the
performance target against which any algorithm is judged. The per-algorithm
bounds are discussed in the context of related work in
Section~\ref{subsec:rw-bo-regret}.

\section{Regret-Aware Black-Box Optimization}
\label{sec:regret-bbo}

% PURPOSE: This section is the intellectual payoff of the chapter.  It
% synthesizes Acts 1 and 2 and positions the project within the literature.
% The reader should finish this section understanding:
%   (a) why regret is a natural and informative measure for BBO,
%   (b) what has already been done at this intersection,
%   (c) what remains open, and
%   (d) how this project advances the state of knowledge.
Regret provides a fine-grained complexity measure in the black-box
setting: rather than
simply asking whether an algorithm finds the optimum within $T$
queries, regret tracks
how suboptimal the queries are throughout the process. This is
particularly relevant
when every evaluation is costly and performing well before the budget
is exhausted
matters in practice.

\subsection{Regret as a Performance Measure for Randomized Search Heuristics}
\label{subsec:regret-as-perf}
% - Reframe the classical runtime analysis question in regret terms:
%     Runtime analysis asks: E[T_opt] = ?
%     Regret analysis asks:  E[R_T^inc] = ? for a fixed budget T
% - These are complementary views of the same underlying process
% - Fixed-budget analysis (§2.3.5) is essentially simple regret analysis;
%   making this explicit is conceptually useful
% - Incumbent cumulative regret captures the "area above the fitness curve",
%   generalizing simple regret to the entire trajectory
% - Implications for algorithm design:
%     A low-simple-regret algorithm is good at pure optimization
%     A low-cumulative-regret algorithm is good at "anytime" optimization
%     These objectives can conflict (exploitation vs. exploration tension
%     reappears at the algorithmic design level)
For randomized search heuristics, regret complements rather than
replaces runtime analysis. Runtime asks when a target is first
reached; regret asks how good the algorithm's queried or retained
solutions are throughout the run. Fixed-budget analysis already moves
in this direction, since expected simple regret is just the expected
final sub-optimality after a budget of $T$
evaluations~\cite{jansen2014fixed,bubeck2011pure}.

The distinction matters operationally. Low simple regret is
appropriate when only the final returned solution matters. Low
\acrshort{cr} is appropriate in anytime settings, where useful
intermediate performance also
matters~\cite{cesa2006prediction,pinto2018practice}. For the
heuristics considered in this project, the incumbent-based
formulation is the natural one because the algorithm's output at
interruption is the best-so-far solution.
